# 1.2 Literature and Practice Review

This chapter reviews how companies and researchers have addressed AI agent reliability, establishing the theoretical foundations for the solution developed in Chapter 2. The review is organized in two parts. Section 1.2.1 examines current industry practice: benchmark evidence for the reliability gap, the economics that make it urgent, and five company cases that introduce the key concepts underpinning the proposed solution. Section 1.2.2 surveys the academic foundations: the ceiling of static prompting, the impracticality of fine-tuning, automated prompt optimization, knowledge distillation, and the superficial alignment hypothesis. Section 1.2.3 synthesizes these observations into five conclusions that directly shape the solution design.

A useful lens for understanding the thesis's core architectural decision comes from dual-process theory in cognitive psychology. @kahneman2011 distinguishes between System 1---fast, automatic, and intuitive processing---and System 2---slow, deliberate, and analytical reasoning. Standard autoregressive language models function analogously to System 1: they produce fluent outputs through pattern recognition but struggle with multi-step planning, self-correction, or policy adherence [@yao2023react]. Reasoning models---such as OpenAI's o1 [@openai2024], DeepSeek-R1 [@deepseek2025], and Kimi K1.5 [@kimi2025a]---attempt to endow LLMs with System 2 capabilities by scaling test-time compute through extended chains of thought trained via reinforcement learning. @snell2024 formally demonstrated that scaling test-time compute can be more effective than scaling model parameters.

This distinction has direct implications for enterprise deployment. In voice AI customer service---the primary application domain for $\tau^2$-bench---agents must respond within 500--800 milliseconds to retain natural conversational flow; latency of one second raises customer abandonment rates by ~40% [@cresta2025; @retellai2025]. Reasoning models improve performance by producing intermediate tokens before a final response, but are therefore fundamentally incompatible with latency requirements. Enterprises require System 2-like capabilities for reliable behavior, but need System 1 latency. What this work attempts is resolving this tension by temporal disjunction: leveraging System 2 at design time (the teacher model analyzing failures and generating prompt patches offline) to improve System 1 performance at run time (the student model responding within latency constraints using improved prompts). This framing positions the contribution as a principled architectural pattern that mirrors how human organizations operate: experts deliberate on procedures so that frontline staff can execute quickly and reliably.

## 1.2.1 How Companies Address AI Agent Reliability

### The enterprise reliability gap

LLMs are most useful as agents, replicating human behavior. As such, their performance on tasks requiring agency has to be somehow measured. A cottage industry of benchmarks arose that attempt exactly this---evaluate LLMs on real-world multi-turn conversations, tool calls, and policy following. This work builds on the $\tau$-bench family. The original 2024 benchmark [@yao2024] measured tool--agent--user interaction in retail and airline customer service; GPT-4o failed 50%+ of the tasks, and the probability of solving the same task 8 times consecutively was <25% in retail domain. $\tau^2$-bench [@barres2025] complicate the tasks by testing tool calls and end database state, and by adding a telecom domain. It is on this benchmark that we will be building in this work.

There are numerous other benchmarks. AgentBench [@liu2023] tested 29 models across 8 various environments and found that long-term reasoning, decision-making, and instruction following were the main bottlenecks. SWE-bench [@jimenez2024] pulled 2,294 software engineering tasks from GitHub; the best model solved under 2% of them. ToolBench [@qin2023] threw over 16,000 real-world RESTful APIs at open-source instruction-tuned models; they scored zero percent, and even ChatGPT needed a bespoke depth-first search decision tree to perform reasonably. API-Bank [@li2023] found that GPT-4 still failed at parameter extraction, API selection, and sequential planning across 73 tools. GAIA [@mialon2023] asked 466 real-world questions requiring reasoning, browsing, and tool use; humans scored 92%, GPT-4 with plugins scored 15%---a 77-point gap running precisely opposite to the usual narrative of LLMs matching or beating human performance. Most recently, CRMArena and CRMArena-Pro [@huang2025crmarena; @huang2025crmarenapro] brought enterprise customer service benchmarking into a realistic Salesforce sandbox with 19 expert-validated tasks across service, sales, and CPQ scenarios. Leading agents achieved only 58% single-turn success, dropping to 35% in multi-turn settings. CRMArena-Pro also found a troubling tension: prompting agents to respect data sensitivity reduced task accuracy, suggesting that enterprise constraints actively conflict with task performance.

Two meta-analyses frame these results. @kapoor2024 analyzed agent benchmarking practices and found 50-fold cost variation for similar accuracy levels; complex agent architectures buy marginal accuracy gains at exponential cost. This supports the argument that prompt-level optimization is a more efficient improvement path than architectural scaling. @rabanser2025 proposed 12 metrics decomposing agent reliability across consistency, robustness, predictability, and safety. Testing 14 models over 18 months on GAIA and $\tau$-bench, they identified a persistent *capability--reliability gap*: accuracy improves faster than reliability. They argue that enterprise autonomous operation requires 3 to 5 nines (99.9%--99.999%); current LLM agents are not on track to reach this threshold through scaling alone. @reliabilitybench2026 formalized this production gap quantitatively, introducing a three-dimensional reliability surface $R(k, \varepsilon, \lambda)$ that unifies consistency (over $k$ trials), robustness (under perturbation level $\varepsilon$), and fault tolerance (at fault intensity $\lambda$). Testing 1,280 episodes across GPT-4o variants, they found that perturbations alone cause 8.8% reliability degradation, and that simpler ReAct architectures outperform Reflexion in surface volume---suggesting that complex self-reflection mechanisms may amplify fault impacts rather than mitigate them. Their practical heuristic is sobering: if a benchmark reports 90% accuracy, expect 70--80% in production once consistency and faults are accounted for.

### Economics of AI agent deployment

The reliability gap acquires urgency when set against the economics of customer service. Fully automated AI interactions cost \$0.50--\$2.00 per interaction versus \$5--\$15 for human agents [@gartner2023cost; @quidget2025], creating a 5--10$\times$ cost advantage that has driven rapid market growth: the contact center AI market is expanding at 21--25% CAGR, from roughly \$2 billion in 2024 to a projected \$7--13 billion by 2030--2034 [@grandviewresearch2024; @fortunebi2025]. Yet the gap between market aspiration and operational reality is wide. The RAND Corporation, based on interviews with 65 data scientists, found that over 80% of AI projects fail---twice the rate of non-AI IT projects---with root causes including misaligned problem framing, inadequate data, and infrastructure gaps [@ryseff2024]. MIT's 2025 report found that 95% of generative AI pilots fail to deliver measurable P\&L impact, with specialized vendor-led projects succeeding approximately 67% of the time versus only 33% for internal builds [@mitnanda2025]. Only 25% of enterprises have moved more than 40% of their AI pilots into production [@deloitte2026ai].

The most prominent case study illustrates both the promise and the fragility. Klarna's AI assistant handled 2.3 million conversations in its first month---equivalent to 700 full-time agents---reducing resolution time from 11 to 2 minutes and projecting \$40 million in annual profit improvement [@klarna2025]. Yet within a year, customer satisfaction had declined, and the company began rehiring human agents, with CEO Sebastian Siemiatkowski acknowledging that the company had "focused too much on efficiency and cost" [@forrester2025regret]. The pattern is not isolated: Gartner has predicted that 50% of organizations will abandon plans to reduce their customer-service workforce through AI by 2027 [@gartner2025abandon], and among companies that did pursue AI-driven layoffs, 55% reported regretting the decision [@forrester2025regret].

The talent required to close the reliability gap through manual prompt engineering is in acute shortage. PwC's 2025 Global AI Jobs Barometer found that workers with AI skills command a 56% wage premium, up from 25% in 2024 [@pwc2025aijobs]. Prompt engineers earn a median of \$126,000--\$128,000, with senior roles reaching \$300,000+ [@glassdoor2025prompt]. IDC projects that over 90% of global enterprises will face critical AI skills shortages by 2026, with sustained gaps risking \$5.5 trillion in losses [@idc2025skills]. Forrester estimates that three out of four firms attempting to build agentic architectures independently will fail [@forrester2025].

The problem is compounded by *agent drift*: unlike traditional software, LLM agents exhibit progressive behavioral degradation over extended deployment even without explicit parameter changes [@agentdrift2025]. @informatica2025 quantify the scope: 91% of AI models experience quality degradation over time. Post-deployment maintenance requires 0.5 to 3 full-time equivalents and \$50,000--\$100,000 per year per deployment [@gartner2025complexity], and enterprise implementations routinely cost 3 to 5 times the initially advertised price [@acceldata2025].

A concurrent pricing shift intensifies the pressure on vendor economics. The industry is moving from per-seat licensing to outcome-based models: Intercom charges \$0.99 per resolution [@intercom2024], Sierra implements pure outcome-based pricing [@sierra2024], and seat-based pricing has dropped from 21% to 15% of companies in 12 months. Under outcome-based pricing, vendors bear the economic risk of agent performance directly---every unresolved interaction is revenue forgone. AI-first companies already operate at 50--60% gross margins, well below the 75--90% typical of traditional SaaS [@bessemer2025]. Gartner has further warned that generative AI cost per resolution will exceed \$3 by 2030 as current LLM vendor pricing---subsidized by up to 90%---normalizes [@gartner2026costwarning].

The scale of the opportunity, however, is commensurate with these challenges. Foundation Capital estimates the "Services as Software" market at \$4.6 trillion [@foundationcapital2024]. McKinsey projects that agentic AI could unlock \$100--400 billion in incremental spending in tech services alone by decade's end [@mckinsey2025techservices]. @brynjolfsson2025, studying 5,172 customer support agents in a paper published in *The Quarterly Journal of Economics*, found that AI assistance increases worker productivity by 14--15% on average and by 34% for novice workers, because AI disseminates best practices of more able workers. This finding directly supports the thesis's approach: a system that continuously learns from best practices can create compounding productivity gains without sustained human intervention.

**Intercom and Sierra** exemplify the business model shift that makes automated agent maintenance a competitive necessity. Intercom's Fin AI Agent, launched in 2024, charges \$0.99 per successful resolution rather than per seat [@intercom2024]; Sierra, founded by former Salesforce CEO Bret Taylor, implements pure outcome-based pricing for its conversational AI platform [@sierra2024]. Under these models, the vendor absorbs the economic cost of every agent failure. The copilot architecture---where AI handles routine interactions and escalates edge cases to humans---is the dominant deployment pattern, but it creates a maintenance bottleneck: every unresolved interaction requires human diagnosis of what went wrong and manual prompt revision. The shift from per-seat to outcome-based pricing transforms agent reliability from a quality concern into a direct margin driver.

**DSPy**, developed at Stanford by @khattab2023, reframes prompt engineering as compilation. Rather than hand-crafting prompt strings, developers define pipelines as declarative modules with learnable parameters---prompts and demonstrations---and a compiler automatically optimizes them against a target metric. An informal study found that the LangChain codebase contains 50 strings exceeding 1,000 characters (hand-crafted prompts), compared to none in DSPy. Compiled pipelines outperform expert-created demonstrations by 5--46% on multi-hop QA and math reasoning. DSPy's significance is in demonstrating that prompt optimization can be productionized: it is used in industry, integrated with multiple LLM providers, and supports composing prompting, fine-tuning, and reasoning techniques within a single pipeline. Its limitation is scope: evaluations remain on single-turn or few-turn reasoning, not multi-turn tool-calling agent interactions.

**Reflexion** [@shinn2023] introduced the concept of verbal self-reflection for agentic improvement: after task failure, the agent generates a natural-language reflection on what went wrong and stores it in episodic memory, improving performance on subsequent attempts. Reflexion produced gains on AlfWorld (household tasks), HotPotQA (multi-hop QA), and HumanEval (code generation). The method demonstrates that automated failure diagnosis and correction is feasible within a single agent---a key premise of this thesis. However, reflections are ephemeral per-episode memory that does not persist across deployments; there is no permanent prompt patch, no separate teacher model, and no evaluation on tool-calling enterprise benchmarks.

**PromptBreeder** [@fernando2023], developed at Google DeepMind, takes an evolutionary approach: it evolves both task-prompts and the mutation-prompts that generate them in a self-referential loop, outperforming Chain-of-Thought and Plan-and-Solve on arithmetic and commonsense reasoning. The evolutionary metaphor---treating prompts as phenotypes subject to selection pressure---validates the approach this thesis adopts at the outer-loop level. But PromptBreeder's testing is confined to single-turn NLU benchmarks (GSM8K, SVAMP), with no involvement of tool calling, multi-turn conversation, or policy adherence.

**Artemis** [@artemis2025] and **PLAY2PROMPT** [@fang2025] represent the most recent wave of tool description optimization. PLAY2PROMPT is a zero-shot framework that iteratively explores tool input--output behaviors via beam search to refine documentation and generate usage examples, improving zero-shot tool performance by 10--30% on the Berkeley Function-Calling Leaderboard and StableToolBench. Artemis is a no-code evolutionary optimization platform that jointly optimizes prompts, tool descriptions, model parameters, and execution settings through semantically-aware genetic operators, achieving 9.3--13.6% improvement on coding and reasoning benchmarks. These works establish that tool descriptions are a viable optimization target. However, neither employs a teacher--student paradigm where a frontier reasoning model diagnoses failures in a weaker student, and neither evaluates on multi-turn customer service benchmarks with domain-specific policies.

### Russian AI market participants

The Russian AI market presents a distinct context shaped by import substitution policies and data sovereignty requirements under Federal Law No. 152-FZ. The major domestic AI platforms---Yandex with its Alice virtual assistant ecosystem and customer service solutions built on proprietary YandexGPT models [@yandex2024], Sber with its GigaChat and SalutDevices platform [@sber2024], and MTS AI with conversational agents for telecom customer service [@mtsai2024]---all face the same agent maintenance challenge documented above. These companies operate large-scale conversational AI deployments serving millions of users, but public documentation of their approaches to post-deployment agent improvement is limited. Industry reports suggest that agent maintenance at Russian AI companies relies primarily on dedicated prompt engineering teams performing manual diagnosis-and-repair cycles [@raec2025], with some adoption of A/B testing frameworks for prompt variants. The sanctions-driven constraints on access to Western frontier models (GPT-4, Claude) make the teacher--student architecture proposed in this thesis particularly relevant: domestic open-weight models can serve as both teachers and students, and the entire optimization pipeline can operate within data sovereignty boundaries. The Russian National AI Strategy to 2030 [@russiaai2024] explicitly targets customer service automation as a priority domain, creating additional institutional demand for cost-effective agent improvement methods.

## 1.2.2 Academic Foundations

### The ceiling of static prompting and scaffolding

It has long become apparent that model performance on the benchmark is not the ceiling it can truly attain. By editing the model input---specifically, its instruction and tools---improvements can be garnered. Additionally, expending more tokens at test time was also shown to be useful.

Chain-of-thought prompting [@wei2022] showed that a few hand-crafted reasoning exemplars can substantially improve reasoning, with PaLM-540B reaching state-of-the-art on GSM8K. The ReAct framework [@yao2023react] interleaved reasoning traces with task-specific actions, letting models call external tools while maintaining reasoning chains; it outperformed both pure chain-of-thought and pure action-generation baselines on question answering, fact verification, and interactive decision-making. @yao2023tot generalized this with Tree of Thoughts, where LMs explore multiple reasoning paths via tree search with self-evaluation, reaching 74% on Game of 24 compared to chain-of-thought's 4%.

On the tool-use front, since @openai2023 introduced function calling, moving interaction from free-form text generation to schema-driven structured output, many have employed this format to produce more valid responses. @schick2023 trained Toolformer in a self-supervised manner to invoke external tools autonomously, matching much larger models at zero-shot performance. @patil2023 fine-tuned LLaMA through Gorilla to surpass GPT-4 on API call generation, while also showing that GPT-4 frequently hallucinates incorrect API usage under prompting alone. @willard2023 tackled format reliability with constrained decoding via finite-state machines, guaranteeing valid structured output but only addressing format---not reasoning, not planning.

For all these advances, static approaches share a basic limitation: they fix the agent's behavioral repertoire at design time. ReAct exemplars are hand-crafted and task-specific. Function schemas consume tokens and need careful engineering. Chain-of-thought reasoning only emerges at roughly 100 billion parameters [@wei2022]. @brown2020 showed that in-context few-shot learning has task-dependent ceilings, with GPT-3 failing on ANLI and QuAC. @zhou2022 found through Automatic Prompt Engineer (APE) that optimal prompts are fragile: small wording changes alter effectiveness.

@sclar2023 provide the sharpest evidence for this ceiling. Meaning-preserving formatting changes in few-shot prompts---changes as trivial as spacing and delimiter choice---produce up to 76 accuracy points of variation on LLaMA-2-13B. Larger models, more examples, and instruction tuning did not eliminate this sensitivity. @Fig:prompt-sensitivity visualizes the magnitude of this effect across models. The consequence for agent deployment is clear: static prompt scaffolds, however carefully designed, cannot guarantee consistent behavior. In business terms, every policy change, new product, or regulatory update requires a human expert to revisit and re-engineer the agent's prompts and tool schemas---a recurring cost compounded by the agent drift phenomenon documented in Section 1.2.1.

### The impracticality of fine-tuning for rapid enterprise adaptation

The previous section established that static prompting hits a ceiling; the natural next question is whether updating model weights can break through it. The answer is yes---but at a cost that makes the approach structurally incompatible with how enterprises need to operate. This section reviews the three principal methods for modifying LLM behavior through weight updates---supervised fine-tuning (SFT), reinforcement learning from human feedback (RLHF), and direct preference optimization (DPO)---alongside parameter-efficient techniques that reduce their cost. All weight-modification methods, regardless of how they differ in mechanism, share three liabilities: high upfront cost, an alignment tax on general capabilities, and irreversibility that compounds with each successive adaptation.

**Supervised fine-tuning** is the most direct form of weight modification: show the model what good outputs look like, and it learns to produce them. @zhou2023lima demonstrated this can work with remarkably little data---fine-tuning LLaMA-65B on just 1,000 carefully curated examples produced alignment quality competitive with RLHF-trained models. Their "Superficial Alignment Hypothesis" holds that alignment primarily teaches style and format rather than injecting new knowledge (discussed further in Section 1.2.2.5). Still, each new enterprise domain would need its own expert-curated dataset, and every time an enterprise's policies change, the dataset must be rebuilt and the model retrained.

**RLHF** goes beyond SFT by training the model to satisfy preferences that are difficult to express as input--output pairs. @ouyang2022 introduced InstructGPT, where a 1.3B-parameter model fine-tuned with RLHF was preferred by human annotators over 175B-parameter GPT-3 in 85% of comparisons. But the pipeline involves three stages---supervised fine-tuning on labeler-written demonstrations, reward model training on ranked outputs, and PPO-based reinforcement learning---each demanding its own dataset, infrastructure, and iteration cycle. The authors identified an "alignment tax": alignment improved preference ratings but caused performance regressions on public NLP benchmarks. @bai2022 proposed Constitutional AI to reduce the human feedback burden, replacing human harmlessness labels with model-generated critiques guided by constitutional principles, but it still requires a complex multi-phase pipeline.

**DPO** [@rafailov2023] eliminates the separate reward model and RL stage by directly optimizing on preference data with a classification loss. DPO matches or exceeds PPO-based RLHF with much less complexity, but the core bottleneck---the need for domain-specific preference data---is unchanged. **LoRA** [@hu2022] reduced fine-tuning costs by injecting trainable low-rank matrices while freezing pretrained weights, cutting trainable parameters by up to 10,000$\times$. LoRA makes SFT, RLHF, and DPO cheaper to execute, but does not eliminate the need for task-specific training data.

**The alignment tax.** The deeper issue with any weight-modification approach is not just cost but irreversibility. @luo2023 found that catastrophic forgetting is universal across model sizes during continual instruction tuning, and counterintuitively, larger models forget more severely. @lin2024 quantified the alignment tax more precisely: RLHF alignment degrades pretrained LLM abilities in translation, reading comprehension, and common-sense reasoning, and existing mitigations do not fully compensate. @casper2023 surveyed RLHF challenges broadly---noisy feedback, reward hacking, distributional shift, sycophancy---and argued that RLHF is not a complete alignment framework. @young2026 provided the first mathematical definition of the alignment tax: under linear representation assumptions, the Pareto frontier governing safety--capability trade-offs admits an *irreducible component* determined by data structure that cannot be eliminated by algorithmic improvements. @shenfeld2026 documented that standard SFT "substantially degrades performance" and "sharply shortens responses, indicating a collapse in reasoning behavior" when applied to reasoning models.

**The case for operating above the weight level.** Across the board, weight-modification approaches impose costs in data collection, compute, and maintenance that make them unsuitable for rapid, reversible, domain-specific agent improvement. @cai2025 proposed Training-Free GRPO, shifting optimization from parameter space to context space entirely, outperforming fine-tuned 32B models while reducing learning costs from roughly \$800 to \$8. The authors explicitly argue that standard GRPO "incurs prohibitive computational costs and risks catastrophic forgetting"---an indictment from within the weight-modification paradigm itself. All of this motivates the search for an alternative that operates at the prompt and tool description layer, where changes are instant, reversible, and composable without risk of degrading the underlying model. @Fig:finetuning-comparison summarizes the trade-offs across methods.

### Automated prompt optimization

Given fine-tuning's limitations, it naturally becomes attractive to optimize prompts instead of weights. This section reviews the main approaches beyond those introduced in the company cases (Section 1.2.1), establishing two claims. First, automated prompt optimization is a mature paradigm with demonstrated gains across a range of tasks. Second, despite this maturity, no method has been validated on structured tool-agent-user benchmarks with enterprise-grade success criteria.

**Gradient-inspired and search-based optimizers.** @yuksekgonul2024 proposed TextGrad, performing automatic "differentiation" via text by backpropagating LLM-generated textual feedback through computation graphs. Published in Nature in 2025, TextGrad represents perhaps the strongest evidence that prompt-level optimization is a serious paradigm: it works across coding (20% relative gain on LeetCode-Hard), science (GPT-4o zero-shot on GPQA improved from 51% to 55%), molecule optimization, and radiotherapy planning. But evaluations are on single-turn tasks, not multi-turn tool-calling benchmarks. @yang2023 developed OPRO, using LLMs as black-box optimizers by describing optimization tasks in natural language, outperforming human-designed prompts by up to 8% on GSM8K and 50% on Big-Bench Hard.

**Evolutionary approaches.** @guo2023 combined LLMs with evolutionary algorithms in EvoPrompt, significantly outperforming human-engineered prompts across 31 datasets---up to 25% on Big-Bench Hard. @pryzant2023 proposed ProTeGi, using LLM-generated natural language "gradients" to iteratively edit prompts, reaching up to 31% improvement on classification tasks. @agarwal2024promptwizard introduced PromptWizard, a self-evolving mechanism where the LLM generates, critiques, and refines its own prompts; it outperformed all baselines (APE, DSPy, OPRO, EvoPrompt, PromptBreeder) on 45 tasks at just \$0.05 per task---a 5--60$\times$ cost reduction compared to competitors.

**Agentic self-improvement.** @hu2024 proposed ADAS, defining agents in code and using a meta-agent to iteratively program new designs from an ever-growing archive. ADAS operates at the level of entire agent architectures rather than iteratively patching prompts. @cheng2024 developed Trace and OptoPrime, framing workflow optimization over execution traces, beating DSPy's COPRO by roughly 10% on Big-Bench Hard. @zhang2024 introduced AgentOptimizer, treating an agent's tools as learnable parameters---the closest to tool-description optimization, but evaluated on MATH and tabular reasoning, not customer service. @wu2024 proposed AvaTaR, using contrastive reasoning to generate prompts for tool-assisted knowledge retrieval.

**Tool description optimization.** Beyond the Artemis and PLAY2PROMPT cases discussed in Section 1.2.1, @guo2026 proposed Trace-Free+, a curriculum learning framework that trains LLMs to rewrite tool descriptions, including parameter schemas, without requiring execution traces at deployment time. Trace-Free+ showed consistent gains on StableToolBench and RestBench with strong cross-domain generalization and robustness even with 100+ candidate tools.

None of these methods have been validated on structured multi-turn tool-calling benchmarks with realistic user simulation, domain-specific policies, and enterprise-grade success criteria.

### From weight-level distillation to prompt-level knowledge transfer

The previous two sections established that static prompting plateaus and weight modification is impractical for enterprise deployment. But there is a third option: use a strong model to improve a weaker one without modifying either model's weights. This section traces the evolution of teacher--student knowledge transfer from weight-level distillation through output-level and instruction-level transfer, showing that each generation trades fidelity for practicality---getting lighter, cheaper, and more reversible.

**Weight-level distillation.** The idea that a strong model can transfer knowledge to a weaker one was formalized by @hinton2015, who trained a smaller "student" network to replicate the soft probability distributions of a larger "teacher" via temperature-scaled softmax. The key insight---"dark knowledge"---is that the teacher's soft targets carry far more information than hard labels. In NLP, @sanh2019 applied distillation during pre-training to produce DistilBERT, 40% smaller than BERT while retaining 97% of its language understanding. @jiao2020 pushed further with TinyBERT, 7.5$\times$ smaller and 9.4$\times$ faster than BERT-Base at 96.8% of its performance. Weight-level distillation requires full retraining of the student---exactly the process that the previous section showed to be impractical for rapid enterprise iteration.

**Output-level distillation.** Large language models shifted the paradigm. @taori2023 fine-tuned LLaMA-7B on 52,000 instruction-following examples generated by text-davinci-003 to create Stanford Alpaca for under 600 US dollars. @chiang2023 fine-tuned LLaMA-13B on roughly 70,000 ChatGPT conversations to create Vicuna, reaching about 90% of ChatGPT quality. @wang2023 formalized this with Self-Instruct, where a model generates its own instruction-following training data through bootstrapping. The medium of transfer changed---from probability distributions to text outputs---but the student's weights are still updated.

**Iterative distillation.** A crucial advance came when researchers made the transfer loop iterative and failure-aware. @jiang2023 proposed Lion, an adversarial distillation framework where the teacher identifies "hard" instructions where the student fails and creates new challenging instructions targeting those weaknesses. Using just 70k examples, Lion-13B surpassed Vicuna-13B by 55.4% on Big-Bench Hard. @xu2023wizard introduced Evol-Instruct through WizardLM, where a strong LLM progressively rewrites simple instructions into more complex ones. @zheng2023 studied the LLM-as-judge paradigm and showed that GPT-4 agrees with human preferences over 80% of the time, validating that a strong model can serve as a diagnostic supervisor. The pattern is clear: the teacher's role is evolving from "provide examples" to "diagnose failures and generate targeted corrections"---the mechanism this thesis adopts, but at the prompt level rather than the weight level.

**Reasoning and test-time compute.** OpenAI's o1 [@openai2024] pioneered training models via reinforcement learning to generate extended chains of thought; on AIME 2024, o1 solved approximately 79% of problems compared to GPT-4's 9%. @deepseek2025 replicated and extended this with DeepSeek-R1, demonstrating that reasoning capabilities can be incentivized through pure RL without supervised fine-tuning on human-labeled reasoning trajectories. Published in Nature, DeepSeek-R1 showed that advanced reasoning patterns---self-reflection, verification, and dynamic strategy adaptation---emerge naturally during RL training. Crucially, DeepSeek-AI demonstrated that reasoning patterns of larger models can be distilled into smaller ones. The Kimi model family is particularly relevant: Kimi K1.5 [@kimi2025a] reported state-of-the-art reasoning matching OpenAI's o1, and its long-to-short (long2short) transfer methodology---using long chain-of-thought techniques to improve short-CoT models---directly parallels this thesis's approach of transferring a thinking model's diagnostic capabilities into prompt patches for a non-thinking model.

**Prompt-level transfer.** An important precedent is SPoT [@vu2022], which showed that soft prompts learned for one task can initialize prompts for new tasks, improving performance across 26 NLP tasks. This establishes that knowledge encoded in prompts---not just in weights---can be reused across contexts. The trajectory is clear: each generation of distillation has gotten lighter, cheaper, and more reversible, moving the locus of knowledge closer to the surface. This thesis takes the next step: the transfer medium is natural language patches to prompts and tool descriptions---the lightest, cheapest, and most reversible form of knowledge transfer in this lineage, and the form most compatible with how enterprises consume AI through APIs.

### The superficial alignment hypothesis

A cross-cutting theoretical insight connects the fine-tuning limitations of Section 1.2.2.2 with the prompt-level transfer of Section 1.2.2.4. @zhou2023lima demonstrated that fine-tuning LLaMA-65B on just 1,000 carefully curated examples produced alignment quality competitive with RLHF-trained models, proposing the "Superficial Alignment Hypothesis": alignment primarily teaches style and format rather than injecting new knowledge; the knowledge is already in the pretrained weights, and fine-tuning just teaches the model how to present it. If alignment is primarily about style and format, then prompt-level interventions---which operate on exactly this surface---should be capable of producing real behavioral changes without touching the weights at all. This hypothesis provides theoretical grounding for the entire thesis: the diagnose-patch-validate framework works precisely because the student models already possess the knowledge required to handle most enterprise tasks; what they lack is the correct behavioral framing, which can be supplied through prompt and tool-schema patches.

## 1.2.3 Synthesis: Five Conclusions for Solution Design

The preceding review of industry practice and academic foundations converges on five conclusions that directly shape the solution designed in Chapter 2.

**Conclusion 1: Prompt-level optimization is feasible and academically validated.** DSPy, TextGrad, EvoPrompt, PromptBreeder, and PromptWizard have collectively demonstrated that prompts and tool descriptions can be improved algorithmically, producing gains of 5--50% across diverse tasks at costs far below fine-tuning. The field has developed the required machinery---optimization loops, LLM-generated feedback, evolutionary operators---and prompt optimization is no longer a speculative idea but a mature paradigm with published results in Nature.

**Conclusion 2: No existing method has been applied to multi-turn tool-calling agents.** Despite the maturity of prompt optimization, every method has been evaluated on single-turn or few-turn benchmarks: GSM8K, HotPotQA, Big-Bench Hard, GPQA, LeetCode, BFCL, StableToolBench. The $\tau$-bench family---which tests multi-turn tool-calling interactions with realistic user simulation, domain-specific policies, and enterprise-grade success criteria---has not been used as an optimization target by any existing method. The optimization tools exist; the benchmarks exist; the two have not met. @Fig:gap-matrix positions the closest related work against five criteria that define this gap; no existing method satisfies all five.

![Research gap positioning matrix. Each row is a closely related method; each column is a criterion required for the contribution of this thesis. Only this work satisfies all five.](figures/fig_lr_06_gap_matrix.png){#fig:gap-matrix}

**Conclusion 3: A teacher--student setup enables knowledge transfer without weight access.** The distillation trajectory from Hinton through Alpaca, Lion, and WizardLM shows that a strong model can reliably supervise a weaker one, with the teacher's role progressively evolving from "provide examples" to "diagnose failures and generate targeted corrections." The superficial alignment hypothesis provides the theoretical basis: if alignment is about style and format, then the teacher's corrections can be encoded as prompt patches rather than weight updates. The most recent work---GEPA [@agrawal2025], SCOPE [@pei2025], LEAP [@choudhury2024]---approaches this intersection from different directions but each leaves the gap open: GEPA evaluates on reasoning benchmarks, not tool-calling; SCOPE lacks a teacher--student paradigm; LEAP uses weight updates rather than prompt optimization.

**Conclusion 4: Benchmark evaluation is the credible validation methodology.** The meta-analyses by @kapoor2024 and @rabanser2025, the formalization by @reliabilitybench2026, and the development of enterprise-focused benchmarks (CRMArena, $\tau^2$-bench) establish that structured benchmark evaluation with standardized metrics is the accepted methodology for validating agent improvement. The reliability surface framework and the "expect 70--80% in production" heuristic further argue that improvements must be demonstrated on benchmarks that approximate real operating conditions, not just toy tasks.

**Conclusion 5: Three patch surfaces are needed---prompt, schema, and preprocessor.** The tool description optimization literature (PLAY2PROMPT, Trace-Free+, Artemis) establishes that tool schemas are a viable and effective optimization target distinct from system prompts. The benchmarking literature shows that agent failures span reasoning, tool use, and policy adherence. No single patch surface can address all failure types: system prompt patches address reasoning and policy violations, tool schema patches address tool misuse, and tool preprocessors provide guardrails for cases resistant to instruction-level intervention. The combination of all three---which no existing method provides---is necessary to maximize the fix rate.

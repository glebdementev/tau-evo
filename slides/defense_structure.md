## PHASE 2: DEFINE THE MISSION

### Slide 4 — Problem, Goal, and Research Questions
- **Problem:** No existing method combines teacher-driven prompt optimization with validation on structured tool-agent-user benchmarks (Fig 2.6 — gap positioning matrix)
- **Goal:** Determine the extent to which automated, teacher-model-driven prompt and tool-schema evolution can improve agent performance on τ²-bench
- **Research sub-questions:**
  1. Which failure modes in multi-turn tool-calling tasks are most responsive to prompt and tool-schema edits, and which resist them?
  2. What is the scaling behaviour of the evolution framework: does fix rate hold as the task pool grows, or does it degrade?
  3. Can prompt-level patches generated by a teacher model close a meaningful fraction of the baseline-to-frontier performance gap, without any weight updates?
- **Objectives → Deliverables table** (same format as previous defense):

| Objective | Deliverable |
|---|---|
| 1. Baseline evaluation of student models on τ²-bench airline domain | Failure mode catalogue across three student models |
| 2. Design diagnose-patch-validate loop with three patch surfaces | Evolution framework (software artefact) |
| 3. Evaluate across increasing task-pool sizes (5, 10, 20 tasks) and three student models | Scaling data and cross-model comparison |
| 4. Measure gap closure (baseline → evolved → frontier) | Primary outcome metric |
| 5. Analyse distribution of successful and unsuccessful patches | Boundaries of prompt-level intervention |

### Slide 5 — Theoretical Framework (Compressed Literature Review)
- Two parallel columns mirroring the tension slide's structure:
  - **Left column: Agentic benchmarks & prompting** — benchmarks quantify the reliability gap (τ-bench, AgentBench, GAIA); static prompting and tool engineering improve capability but not adaptability; fine-tuning is impractical for rapid iteration (alignment tax, catastrophic forgetting, irreversibility)
  - **Right column: Automated optimisation & distillation** — prompt optimisation methods (DSPy, TextGrad, GEPA) produce gains on reasoning/classification tasks; teacher–student distillation has evolved from weights (Hinton 2015) through outputs (Alpaca, Vicuna) toward prompt-level transfer (SPoT); none validated on tool-calling benchmarks
- Six-step argument chain (Fig 2.1, slide layout version) shown as a compact visual below the columns — the figure does the work, no bullet-point dump
- Single slide. Speaker notes carry the detail on individual papers.

## PHASE 3: SHOW THE WORK

### Slide 6 — Benchmark and Dataset
- τ²-bench (Barres et al., 2025): multi-turn conversations with simulated user, tool calls against simulated database, domain-specific policies, strict binary pass¹ evaluation
- Airline domain used for all experiments: 50 tasks (subsets of 5, 10, 20 selected)
- Three experimental conditions (Fig 3.5):
  - **Condition B (Baseline):** student model, default prompt, default tools → performance floor
  - **Condition K (Evolved):** student model, evolved prompt, patched tools → intervention
  - **Condition F (Frontier):** teacher model, default prompt, default tools → performance ceiling
  - Gap Closure = (K − B) / (F − B) × 100%

### Slide 7 — Model Selection
- Table (same format as previous defense):

| Role | Model | Rationale |
|---|---|---|
| Student agent (primary) | Qwen3 30B-A3B (3.3B active params) | Cost-efficient MoE; non-trivial τ²-bench scores but meaningful headroom to frontier |
| Student agent (ablation) | Qwen3.5 Flash | Stronger instruction follower; tests whether student capability changes the framework's yield |
| Student agent (ablation) | GLM 4.7 Flash (Zhipu AI, 9B params) | Different architecture family; tests generalisability and reveals failure modes |
| Teacher | Kimi K2.5 (Moonshot AI, ~1T MoE) | Frontier-class reasoning in thinking mode; 256K context; open-weight; architecturally independent from students |
| User simulator | Qwen3 30B-A3B | Follows scripted scenarios; no frontier capability needed |

### Slide 8 — The Evolution Framework
- System architecture diagram (Fig 3.3, slide layout)
- Two nested loops:
  - **Outer loop** (Fig 3.1): Evaluate all tasks → extract failures → fix in parallel (teacher sessions) → merge via merger LLM → re-evaluate. Repeats for up to 3 sweeps.
  - **Inner loop** (Fig 3.2): Per-failure fix with two-phase escalation — Phase 1 (teaching): patch_prompt + patch_tool + read_tool_code; Phase 2 (guardrails): + patch_tool_code. Strict validation: unanimous pass across all trials.
- Three patch surfaces (Fig 3.6): prompt patches (behavioural rules), tool schema patches (parameter descriptions, constraint notes), tool preprocessors (input coercion, format normalisation)

### Slide 9 — Evaluation Metrics and Failure Taxonomy
- Primary metric: pass¹ — fraction of tasks achieving perfect reward of 1.0. Binary; no partial credit.
- Gap closure formula: G = (K − B) / (F − B) × 100%
- Fix success rate: fraction of attempted fixes that produce unanimous pass
- Failure taxonomy (Fig 3.8): TOOL_MISUSE, POLICY_VIOLATION, REASONING_ERROR, COMMUNICATION_ERROR — classified automatically from teacher diagnosis; drives per-category analysis
- Fix tier classification: instruction (prompt/schema only, Phase 1), tools (schema-only patches at 20-task scale), guardrail (preprocessor code, Phase 2)

## PHASE 4: DELIVER THE EVIDENCE

### Slide 10 — Finding 1: Prompt evolution produces measurable improvement across all three scales
- **One claim:** The framework improves pass rate for Qwen3 30B-A3B at every scale tested, with stable absolute gain at 5–10 tasks and diminishing returns at 20
- **One visual:** Cross-scale summary table (Table 32) — 5 tasks: +20pp; 10 tasks: +23pp; 20 tasks: +11pp. Fix rate: 100% → 56% → 53%
- Speaker notes carry the per-task heatmaps (Figs 22, 28, 37) as backup

### Slide 11 — Finding 2: The framework's value is contingent on student model capability
- **One claim:** Three student models receive identical teacher patches; outcomes diverge sharply
- **One visual:** Cross-model comparison table at 10 tasks (Table 22) — Qwen3.5 Flash: +30pp majority, 80% fix rate; Qwen3 30B-A3B: +20pp, 71%; GLM 4.7 Flash: 0pp, 0% fix rate on genuinely failing tasks
- At 20 tasks: Qwen3.5 Flash reaches 65% majority (+20pp); Qwen3 30B-A3B reaches 30% (+5pp). GLM dropped due to negative results at 10.
- Speaker notes: GLM's continuous degradation (50% → 43% → 40%) motivating its exclusion

### Slide 12 — Finding 3: 70–92% of successful fixes are instruction-tier patches
- **One claim:** Instruction-level patches (prompt text + schema edits) dominate across all experiments; ratio is stable regardless of scale or student model
- **One visual:** Fix tier breakdown across experiments — 71/29 at 5 tasks, 71/29 at 10 (Qwen3 30B-A3B), 80/20 at 10 (Qwen3.5 Flash), 70/20/10 at 20 tasks (with tool-schema tier emerging)
- Supports the Superficial Alignment Hypothesis (Zhou et al., 2023)
- Speaker notes: tool-schema fixes appear only at 20 tasks — larger failure surface exposes modes not addressable by instruction alone

### Slide 13 — Finding 4: A hard core of resistant tasks emerges and is consistent across models
- **One claim:** Tasks 7, 9 resist all fix attempts across all three student models. The unfixable set shrinks with stronger students but does not disappear.
- **One visual:** 10-task heatmap (Fig 28) showing the bifurcation — left cluster greens progressively, right cluster stays red
- Qwen3.5 Flash fixes Tasks 11, 12 that Qwen3 30B-A3B cannot — these were capability-limited, not structurally hard
- Speaker notes: wasted effort stats (800+ messages, 7+ hours at 20 tasks)

### Slide 14 — Finding 5: Patch interference causes regression; severity varies by model architecture
- **One claim:** Accumulated patches create conflicting directives, causing regression that parallels catastrophic forgetting in weight space
- **One visual:** GLM 4.7 Flash at 5 tasks — sweep 2 reaches 80% majority, sweep 3 collapses to 40% (Fig 25/26)
- Qwen3 30B-A3B: mildest regression. Qwen3.5 Flash: severe at 10 tasks (-17pp trial), stabilises at 20. GLM: monotonic decline at all scales.
- Speaker notes: Task 11 on Qwen3.5 Flash as case study — fixed in 42 seconds, undone by unrelated sweep 2 patches

### Slide 15 — Hypothesis Summary (Scoreboard)
- Research sub-questions with verdicts (checkmark/X format matching previous defense):

| Sub-question | Result | Status |
|---|---|---|
| 1. Which failure modes respond to prompt/tool-schema edits? | Policy comprehension failures most responsive (70–92% instruction-tier). TOOL_MISUSE addressable via schema patches. Multi-step reasoning failures resist all intervention. Hard core is consistent across models. | ✓ (with nuance) |
| 2. Does fix rate hold as the task pool grows? | Absolute improvement stable at 5–10 tasks (+20pp), halves at 20 tasks (+11pp). Fix rate declines 100% → 56% → 53%. Framework reaches capacity ceiling of 5–8 fixable tasks regardless of pool size. | ✓ (diminishing returns confirmed) |
| 3. Can prompt-level patches close a meaningful fraction of the gap? | Yes for capable students: Qwen3.5 Flash reaches 65% majority at 20 tasks (+20pp). Not for weaker instruction followers: GLM 4.7 Flash achieves zero net improvement. Framework's value is contingent on student model capability. | Partial ✓ |

## PHASE 5: LAND THE PLANE

### Slide 16 — Conclusions and Contributions
- Four contributions (matching thesis §1.8):
  1. A teacher-driven prompt evolution framework operating entirely in the input space of a frozen student model, with three patch surfaces and validation before merging
  2. First empirical evaluation of automated prompt optimisation on a multi-turn, tool-calling benchmark (τ²-bench)
  3. Empirical evidence on scaling behaviour: stable absolute gain, declining fix rate, emergence of a hard core, and the role of student model capability
  4. Indirect support for the Superficial Alignment Hypothesis in agentic settings — 70–92% of fixes are simple instruction-level patches
- Limitations (two columns, matching previous defense format):
  - Single domain (airline); retail and telecom untested
  - Single teacher model; sensitivity to teacher choice unknown
  - 3 trials per task; limited statistical power
  - No comparison against fine-tuning baselines (RLHF/DPO)
  - No patch retirement mechanism; cumulative interference
  - Best result (65% majority) remains far from enterprise-grade reliability (99.9%+)
